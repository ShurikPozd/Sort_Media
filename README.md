# 📸 Сортировщик фото и видео iPhone (и других устройств)

Скрипт на Python, который автоматически сортирует файлы в указанной папке:

- **Фотографии** определённой модели камеры (например, iPhone 14 Plus, Samsung Galaxy S23) – в отдельную папку.
- **Видео Live Photo** (связанные с этими фото) – в специальную папку.
- **Остальные видео** (не Live, без модели или не распознанные) – в отдельную папку.
- **Медиа (фото и видео) с других камер**, у которых удалось определить модель → в отдельную папку с созданием подпапок по названию модели (например, `other_cameras/iPhone 7/`, `other_cameras/SM-G991B/`).
- **Все прочие файлы** – в папку для «остального».
- **Работает как с перемещением, так и с копированием**.
- **Поддерживает HEIC/HEIF** (современный формат iPhone).

## 🧰 Подготовка фото и видео

Чтобы выгрузить фотографии и видео с iPhone на компьютер, **рекомендуется использовать программу [3uTools](https://www.3u.com/)** – она позволяет копировать медиафайлы без потери метаданных (EXIF, модель камеры, дата съёмки) и сохранять структуру в одну папку.

**Краткая инструкция:**

1. Подключите iPhone к компьютеру, откройте 3uTools.
2. Перейдите в раздел **«Photos»** → **«Camera Roll»** (или нужный альбом).
3. Выделите нужные фото/видео (можно все).
4. Нажмите **«Export»** и выберите **«Export to PC»**.
5. Укажите **исходную папку**, которую вы будете обрабатывать скриптом.

Альтернативно можно просто скопировать папку `DCIM` с iPhone через Проводник, но 3uTools сохраняет имена файлов и метаданные надёжнее.

## 🚀 Возможности

- Запрос модели устройства (можно указать часть названия, например `iPhone 14` или `SM-G991`).
- Автоматическое определение Live Video по совпадению имени с фото.
- Перемещение или копирование (на выбор).
- Можно **пропустить любую категорию** – просто оставьте поле ввода для папки пустым.
- Запоминает уже обработанные файлы (файл `processed.txt`), повторный запуск не дублирует.
- Исключает из обхода целевые папки (если они находятся внутри исходной).
- Подробная статистика по каждой категории.

## 🔍 Как узнать модель вашего устройства

Чтобы скрипт правильно отбирал нужные фото, вам нужно знать точную модель камеры. Вот три способа её узнать:

### 📱 iPhone
1. Откройте **«Настройки»** → **«Основные»** → **«Об этом устройстве»**.
2. Найдите строку **«Название модели»** – там будет что-то вроде *iPhone 14 Plus*, *iPhone 13 Pro Max* и т.д.
3. Именно эту строку (или её часть) нужно будет ввести при запросе скрипта.

### 🤖 Android
1. Откройте **«Настройки»** → **«О телефоне»** (или **«Об устройстве»**).
2. Найдите пункт **«Модель»** (иногда **«Номер модели»**). Например, *SM-G991B*, *Xiaomi Mi 9T*.
3. Вводить можно полное название или часть (например, `SM-G991`).

### 🖥️ Уже скопированное фото на компьютере (Windows)
1. Найдите любое фото, сделанное на вашем устройстве.
2. Щёлкните по нему правой кнопкой мыши → **«Свойства»**.
3. Перейдите на вкладку **«Подробно»**.
4. Найдите поле **«Модель камеры»** (или **«Камера»**, **«Модель»**). Скопируйте это значение.
5. При вводе в скрипт можно использовать полную строку или её часть (регистр не важен).

> 💡 **Пример**: если в свойствах фото указано `iPhone 14 Plus`, вы можете при запросе модели ввести `iPhone 14` – скрипт найдёт все фото с `iPhone 14`, `iPhone 14 Pro`, `iPhone 14 Plus` и т.п.

## 🛠 Установка

1. **Убедитесь, что установлен Python 3.8+**.

2. **Склонируйте репозиторий** (или скачайте ZIP):
   ```bash
   git clone https://github.com/ShurikPozd/Sort_Media.git
   cd Sort_Media
   ```

3. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Запустите скрипт**:
   ```bash
   python sort_media.py
   ```

## 📖 Использование

При запуске скрипт последовательно запросит:

1. **Исходная папка** – где лежат все ваши файлы (подпапки также обрабатываются).
2. **Модель устройства** – например, `iPhone 14 Plus`, `SM-G991B`. Можно указать часть названия.
3. **Папка для фото** (можно оставить пустым, чтобы пропустить).
4. **Папка для видео Live Photo** (можно пустым).
5. **Папка для остальных видео** (можно пустым).
6. **Папка для прочих файлов** (можно пустым).
7. **Папка для медиа с других камер** (можно пустым). Если указана, то все фото и видео, у которых удалось определить модель, не совпадающую с целевой, будут перемещены в подпапки по модели внутри этой папки.
8. **Действие** – `move` (переместить) или `copy` (скопировать).

Скрипт просканирует исходную папку, обработает файлы по правилам и выведет статистику.

> ⚠️ **Важно**: при выборе `move` исходные файлы удаляются. Убедитесь, что у вас есть резервная копия важных данных.

## 📝 Примечания

- Поддерживаемые расширения: `.jpg`, `.jpeg`, `.png`, `.heic`, `.heif`, `.mov`, `.mp4`, `.avi`, `.mkv`.
- Live‑видео определяются по совпадению имени с фотографией (например, `IMG_1234.HEIC` и `IMG_1234.MOV`).
- Все обработанные файлы записываются в `processed.txt`. При повторном запуске они игнорируются. Для повторной обработки удалите этот файл.
- Если после запуска скрипт сообщает, что не найдено новых файлов – значит, всё уже отсортировано.
- Целевые папки могут находиться внутри исходной – скрипт их не трогает.
